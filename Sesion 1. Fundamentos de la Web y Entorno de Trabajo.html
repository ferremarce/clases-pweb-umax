📚 2. Contenido Teórico: Fundamentos Web y Estructura HTML (90 min)

🌐 2.1 Fundamentos de Internet y la Web

🔹 ¿Qué es Internet?

Es una red global de redes de computadoras interconectadas que utilizan el protocolo TCP/IP para comunicarse. No es lo mismo que la Web.

Analogía docente: Internet es como la red de carreteras y cables que conecta ciudades. La Web es uno de los servicios que circula por ella (como el servicio de mensajería).

Datos clave: Nació en 1969 como ARPANET. Hoy conecta más de 5 mil millones de usuarios.

🔹 World Wide Web (WWW)

Sistema de información basado en hipertexto que funciona sobre Internet. Utiliza:

  • HTTP/HTTPS: Protocolo de transferencia de hipertexto.
  • URL: Localizador uniforme de recursos (direcciones web).
  • HTML: Lenguaje de marcado para estructurar contenido.

Historia: Creada por Tim Berners-Lee en 1989 en el CERN. La primera página web fue info.cern.ch.

🔹 Modelo Cliente-Servidor

Cliente: Navegador (Chrome, Firefox, Safari) que realiza solicitudes.
Servidor: Computadora que almacena y entrega archivos (Apache, Nginx, IIS).
Proceso: Solicitud HTTP → Servidor procesa → Respuesta con archivos → Navegador renderiza.

🔹 Estándares Web (W3C)

World Wide Web Consortium, fundado por Tim Berners-Lee, establece recomendaciones para garantizar que los sitios funcionen en cualquier navegador y dispositivo. HTML5, CSS3 y las APIs de JavaScript son estándares W3C.

📄 2.2 Introducción a HTML (HyperText Markup Language)

🔹 ¿Qué es HTML?

HTML no es un lenguaje de programación, es un lenguaje de marcado. Utiliza etiquetas (tags) para describir la estructura y el contenido de una página web.

Analogía: HTML es como el esqueleto de un edificio: define dónde van los muros (estructura), las habitaciones (secciones) y los elementos (contenido).

🔹 Evolución de HTML
  • HTML 1.0 (1991): Primeras versiones muy básicas.
  • HTML 4.01 (1999): Amplia adopción, separación de estilo con CSS.
  • XHTML (2000): Versión más estricta basada en XML.
  • HTML5 (2014): Revolucionario. Aporta etiquetas semánticas (<header>, <nav>, <article>), multimedia nativa (<video>, <audio>), APIs avanzadas.

🏗️ 2.3 Estructura Fundamental de un Documento HTML5

<!DOCTYPE html> ← Declaración del tipo de documento (HTML5)
<html lang="es"> ← Elemento raíz, atributo lang define idioma
  <head> ← Metadatos (no visible en el navegador)
    <meta charset="UTF-8"> ← Codificación de caracteres
    <title>Mi primera página</title> ← Título de la pestaña
    <meta name="description" content="..."> ← Descripción para SEO
  </head>
  <body> ← Contenido visible para el usuario
    <!-- Aquí va todo el contenido -->
    <h1>Título principal</h1>
    <p>Párrafo de texto.</p>
  </body>
</html>
Elemento / EtiquetaFunciónDetalles para el docente
<!DOCTYPE html>Declara la versión de HTMLDebe ser la primera línea. Indica al navegador que use el modo estándar (no quirks mode).
<html>Raíz del documentoContiene todo el documento. El atributo lang es importante para accesibilidad y SEO.
<head>Metadatos del documentoTodo lo que está aquí no se muestra directamente, pero es crucial: título, codificación, enlaces a CSS/JS, descripciones.
<meta charset="UTF-8">Codificación de caracteresPermite usar caracteres especiales (ñ, tildes, emojis). Siempre debe incluirse.
<title>Título de la páginaAparece en la pestaña del navegador y en resultados de búsqueda. Es obligatorio.
<body>Cuerpo visibleTodo el contenido que ve el usuario: textos, imágenes, formularios, etc.
<!-- comentario -->ComentariosNo se muestran al usuario. Útiles para documentar código.

📝 2.4 Etiquetas de Contenido: Encabezados, Párrafos y Listas

🔹 Encabezados (h1 a h6)

<h1>Título principal de la página</h1> ← Solo uno por página (importante para SEO)
<h2>Subtítulo o sección principal</h2>
<h3>Subsección dentro de h2</h3>
<h4>Detalle menor</h4>
<h5>Jerarquía profunda</h5>
<h6>El nivel más bajo</h6>
📌 Jerarquía correcta: Enfatizar que la jerarquía de encabezados debe ser lógica. No se debe saltar de h1 a h3 sin un h2 intermedio. Los motores de búsqueda y lectores de pantalla utilizan esta estructura.

🔹 Párrafos y formato de texto

<p>Este es un párrafo de texto. Los navegadores añaden automáticamente margen superior e inferior.</p>

<p>Puedes usar <strong>texto en negrita con importancia semántica</strong> y <em>texto en cursiva con énfasis</em>.</p>
<p>Para saltos de línea dentro del mismo párrafo, usa <br>.</p>
EtiquetaSignificadoUso recomendado
<strong>Importancia fuerteContenido crítico, advertencias, palabras clave importantes.
<em>ÉnfasisPalabras que deben ser enfatizadas al leer.
<b>Negrita visualSolo estilo, sin significado semántico (no recomendado en HTML5, usar CSS).
<i>Cursiva visualSimilar a <b>, sin significado semántico.

🔹 Listas: ordenadas, no ordenadas y de definición

Lista no ordenada (ul) - viñetas
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

Resultado: • HTML • CSS • JavaScript

Lista ordenada (ol) - numeración
<ol>
  <li>Planificar estructura</li>
  <li>Escribir HTML</li>
  <li>Aplicar CSS</li>
</ol>

Resultado: 1. Planificar estructura 2. Escribir HTML 3. Aplicar CSS

Listas anidadas (estructura jerárquica)
<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
      <li>JavaScript</li>
    </ul>
  </li>
  <li>Backend
    <ul>
      <li>Python</li>
      <li>Node.js</li>
    </ul>
  </li>
</ul>

Nota pedagógica: Las listas anidadas son excelentes para enseñar estructura jerárquica y correcta indentación.

🏛️ 2.5 Elementos Estructurales Semánticos (Introducción)

Etiquetas semánticas (HTML5)
  • <header>: Cabecera de página o sección.
  • <nav>: Menú de navegación.
  • <main>: Contenido principal único.
  • <section>: Agrupa contenido temático.
  • <article>: Contenido independiente (blog, noticia).
  • <aside>: Contenido complementario (sidebar).
  • <footer>: Pie de página.
Ejemplo de estructura semántica básica:
<body>
  <header>Logo + título</header>
  <nav>Enlaces</nav>
  <main>
    <article>Contenido principal</article>
  </main>
  <footer>Pie de página</footer>
</body>
🧠 Reflexión para estudiantes: "Antes se usaban <div> para todo. Las etiquetas semánticas hacen que tu código sea más legible, accesible y mejor posicionado en buscadores."

✅ 2.6 Buenas Prácticas y Herramientas

  • Indentación correcta: El código anidado debe tener sangría (2 o 4 espacios) para facilitar lectura.
  • Atributo alt en imágenes: Fundamental para accesibilidad (lectores de pantalla).
  • Validación W3C: Usar validator.w3.org para verificar que el HTML cumple estándares.
  • Comentarios: Documentar secciones complejas con <!-- comentario -->.
⚠️ Errores comunes que los estudiantes cometen:
  • Olvidar cerrar etiquetas (<p> sin </p>).
  • Anidación incorrecta (ej: <strong><em>texto</strong></em>).
  • Usar etiquetas obsoletas como <font> o <center>.
  • No incluir meta charset, causando problemas con tildes.
🛠️ 3. Configuración del Entorno: Visual Studio Code + Extensiones (30 min)

✅ Paso a paso para el docente

  1. Descarga e instalación de VS Code: code.visualstudio.com.
  2. Extensiones obligatorias:
    • Live Server (Ritwick Dey) - servidor local con autorecarga.
    • Prettier - Code formatter - formateo automático al guardar.
    • Auto Rename Tag - renombra etiquetas de cierre automáticamente.
  3. Configurar Prettier: Archivo > Preferencias > Configuración, buscar "format on save" y activar.
  4. Crear carpeta de trabajo: "MiPrimerSitioWeb".
  5. Abrir carpeta en VS Code y crear archivo index.html.
💡 Demostración en vivo: Muestre cómo usar Emmet: escribir "!" y presionar Tab genera la estructura completa de HTML5 automáticamente.
⌨️ 4. Práctica Guiada: "Mi Primera Página Web" (90 min)

📝 Ejercicio completo

  1. En VS Code, crear index.html y generar estructura con "!" + Tab.
  2. Modificar <title> con "Mi Portafolio - [Nombre]".
  3. Dentro de <body>, construir una página personal que incluya:
<header>
  <h1>Bienvenido a mi portafolio personal</h1>
  <p>Estudiante de Desarrollo Web</p>
</header>

<main>
  <section>
    <h2>Sobre mí</h2>
    <p>Me llamo [Nombre], tengo [edad] años y estoy aprendiendo a crear sitios web. Me apasiona <strong>la tecnología</strong> y <em>resolver problemas</em>.</p>
  </section>

  <section>
    <h2>Mis habilidades</h2>
    <ul>
      <li>HTML5 (aprendiendo)</li>
      <li>CSS3 (próximamente)</li>
      <li>JavaScript (próximamente)</li>
    </ul>
  </section>

  <section>
    <h2>Mis metas en el curso</h2>
    <ol>
      <li>Aprender a maquetar sitios profesionales</li>
      <li>Crear mi portafolio interactivo</li>
      <li>Publicar mi primer sitio en GitHub Pages</li>
    </ol>
  </section>
</main>

<footer>
  <p>© 2025 - Mi primera página web</p>
</footer>

4. Guardar y abrir con Live Server. Verificar que todo funciona.

5. Experimentar: cambiar colores con CSS inline (opcional para avanzados) o agregar una imagen.
