Estructura de una Tabla

Las tablas en HTML5 tienen una estructura semántica:

<table>
    <thead>
        <tr>
            <th>Encabezado 1</th>
            <th>Encabezado 2</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Dato 1</td>
            <td>Dato 2</td>
        </tr>
    </tbody>
</table>

Etiquetas de tabla:

  • <table> - Contenedor principal
  • <tr> - Fila (table row)
  • <th> - Celda de encabezado
  • <td> - Celda de datos

Partes de una Tabla

HTML5 divide las tablas en tres secciones:

<thead> - Encabezado

Define la fila de encabezados. Usualmente contiene <th>.

<tbody> - Cuerpo

Contiene las filas de datos principales.

<tfoot> - Pie

Resumen o totales. Siempre aparece después de tbody.

<table>
    <thead><tr><th>Producto</th><th>Precio</th></tr></thead>
    <tbody><tr><td> Laptop </td><td> $999 </td></tr></tbody>
    <tfoot><tr><td> Total </td><td> $999 </td></tr></tfoot>
</table>

Combinar Celdas

Usa colspan y rowspan para expandir celdas:

colspan - Columnas

Une celdas horizontalmente:

<td colspan="2">Esta celda ocupa 2 columnas</td>

rowspan - Filas

Une celdas verticalmente:

<td rowspan="2">Esta celda ocupa 2 filas</td>

Ejemplo completo:

<table border="1">
    <tr>
        <th colspan="2">Horario Semanal</th>
    </tr>
    <tr>
        <td>Lunes</td>
        <td>Matemáticas</td>
    </tr>
</table>

Accesibilidad en Tablas

El atributo scope ayuda a lectores de pantalla:

<thead>
    <tr>
        <th scope="col">Producto</th>
        <th scope="col">Precio</th>
    </tr>
</thead>
<tbody>
    <tr>
        <th scope="row">Laptop</th>
        <td>$999</td>
    </tr>
</tbody>
  • scope="col" - Encabezado de columna
  • scope="row" - Encabezado de fila

Introducción a Formularios

Los formularios permiten recopilar datos del usuario:

<form action="procesar.php" method="POST">
    <label for="nombre">Nombre:</label>
    <input type="text" id="nombre" name="nombre">
    
    <button type="submit">Enviar</button>
</form>

Atributos de <form>:

  • action - URL del script que procesa los datos
  • method - Método HTTP (GET o POST)
  • enctype - Tipo de codificación (para subir archivos)

La Etiqueta <label>

Los labels son esenciales para accesibilidad:

Método 1: for + id

<label for="email">Email:</label>
<input type="email" id="email">

Método 2: anidado

<label>
    Nombre:
    <input type="text">
</label>
⚠️ Importante: Siempre usa labels. Los usuarios no pueden hacer clic en el texto sin él.

Tipos de Input

HTML5 ofrece muchos tipos de campos:

Texto

Contraseña

Oculta caracteres

Número

Fecha

Elementos de Formulario

<textarea> - Área de texto

<textarea name="comentario" rows="4" cols="50"></textarea>

<select> - Lista desplegable

<select name="pais">
    <option value="py">Paraguay</option>
    <option value="ar">Argentina</option>
</select>

<button> - Botón

<button type="submit">Enviar</button>

Validación con HTML5

HTML5 incluye validación incorporada:

Atributos de validación:

  • required - Campo obligatorio
  • minlength - Mínimo de caracteres
  • maxlength - Máximo de caracteres
  • min - Valor mínimo (números)
  • max - Valor máximo (números)
  • pattern - Expresión regular
  • placeholder - Texto de ejemplo
<input type="email" required placeholder="tu@email.com">
<input type="password" minlength="8" required>
<input type="number" min="1" max="100">

Agrupación: fieldset y legend

Agrupa campos relacionados visualmente:

<form>
    <fieldset>
        <legend>Información Personal</legend>
        <label>Nombre:</label>
        <input type="text" name="nombre">
        <label>Apellido:</label>
        <input type="text" name="apellido">
    </fieldset>

    <fieldset>
        <legend>Preferencias</legend>
        <input type="checkbox" id="news">
        <label for="news">Recibir noticias</label>
    </fieldset>
</form>
💡 Beneficios: Mejor organización visual, accesibilidad mejorada para lectores de pantalla.

Métodos GET vs POST

Elige el método adecuado para tu formulario:

GET

  • Datos visibles en la URL
  • Para búsquedas, filtros
  • Limitado en longitud (~2000 chars)
  • No usar para datos sensibles

POST

  • Datos en el cuerpo de la petición
  • Para enviar datos sensibles
  • Sin límite de tamaño
  • Más seguro
⚠️ Regla: Usa POST para formularios que envíen datos al servidor. Usa GET solo para búsquedas.

Ejemplo: Formulario de Registro

<form action="registro.php" method="POST">
    <fieldset>
        <legend>Datos de Cuenta</legend>
        
        <label for="usuario">Usuario:</label>
        <input type="text" id="usuario" name="usuario" required>
        
        <label for="email">Email:</label>
        <input type="email" id="email" name="email" required>
        
        <label for="pass">Contraseña:</label>
        <input type="password" id="pass" name="pass" minlength="8" required>
    </fieldset>

    <button type="submit">Registrarse</button>
</form>

Buenas Prácticas

✅ Hacer:

  • Usar labels asociados a cada input
  • Agrupar campos relacionados con fieldset
  • Usar type correcto para cada campo
  • Agregar required a campos obligatorios
  • Proporcionar placeholder útil
  • Usar POST para datos sensibles

❌ Evitar:

  • Campos sin label
  • Usar solo mayúsculas o minúsculas
  • Olvidar el atributo name
  • No validar datos importantes
  • Usar GET para contraseñas
