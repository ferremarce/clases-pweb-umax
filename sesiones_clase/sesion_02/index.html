📌 Nota: HTML5 introduce etiquetas semánticas que mejoran la estructura y accesibilidad de las páginas web.

Repaso: Estructura Básica HTML

Todo documento HTML tiene una estructura básica que debemos recordar:

<!-- Documento HTML5 -->
<!DOCTYPE html>
<html lang="es">

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Título de la Página</title>
</head>

<body>
    <!-- Contenido visible -->
</body>

</html>
💡 Dato clave: El DOCTYPE indica al navegador que usamos HTML5, el estándar actual.

Etiquetas Semánticas de HTML5

Las etiquetas semánticas describen el significado del contenido, no solo su apariencia:

🎯 Beneficios

  • Mejor accesibilidad
  • SEO mejorado
  • Código más legible
  • Mantenimiento fácil

⚠️ Antigua forma

Usar solo <div> para todo, sin significado semántico.

Etiquetas estructurales principales:

  • <header> - Encabezado de página o sección
  • <nav> - Navegación principal
  • <main> - Contenido principal
  • <section> - Sección temática
  • <article> - Contenido independiente
  • <aside> - Contenido relacionado
  • <footer> - Pie de página

Estructura Semántica Completa

Veamos cómo se relacionan las etiquetas semánticas en una página:

<body>
    <header>
        <h1>Mi Sitio Web</h1>
        <nav>
            <a href="#">Inicio</a>
            <a href="#">Nosotros</a>
        </nav>
    </header>

    <main>
        <section>
            <article>
                <h2>Artículo Principal</h2>
                <p>Contenido del artículo...</p>
            </article>
        </section>
    </main>

    <footer>
        <p>© 2024 Mi Sitio Web</p>
    </footer>
</body>
⚠️ Importante: Solo debe haber UN <body>, UN <header> principal y UN <main> por página.

Encabezados (h1 - h6)

Los encabezados definen la jerarquía del contenido. Van del 1 al 6:

<h1>Encabezado Principal (solo uno por página)</h1>
<h2>Sección Importante</h2>
<h3>Subsección</h3>
<h4>Subsección Menor</h4>
<h5>Detalle Pequeño</h5>
<h6>Nivel Más Bajo</h6>

Reglas de uso:

  • Solo un <h1> por página (el título principal)
  • No saltarse niveles (no pasar de h2 a h4)
  • Los motores de búsqueda usan los encabezados para entender la estructura

Párrafos y Texto

Las etiquetas básicas para texto:

<p>Este es un párrafo de texto.</p>

<p>Este texto tiene 
    <strong>negrita</strong> 
    y <em>cursiva</em>.</p>

<p>Texto con 
    <u>subrayado</u>, 
    <s>tachado</s>, 
    <mark>resaltado</mark>.</p>

<p>Una línea horizontal:</p>
<hr>

<p>Salto de línea:</p>
<br>
<p>Sigue en nueva línea</p>

<p>Esta es una <abbr title="HyperText Markup Language">HTML</abbr>.</p>

<blockquote cite="https://ejemplo.com">
    <p>Una cita importante que destaca un concepto.</p>
</blockquote>

<p>El libro <cite>El Gran Gatsby</cite> es un clásico.</p>

Etiquetas adicionales de texto:

  • <abbr title="">: Abreviatura con título completo (accessibility)
  • <blockquote cite="">:Cita larga de otra fuente
  • <cite>: Título de una obra (libro, película, etc.)

Estilos Inline con CSS:

Los estilos inline se aplican directamente en el atributo style de cada etiqueta HTML:

<p style="color: blue; padding: 10px;">Texto azul con relleno</p>

<p style="border: 2px solid red; padding: 15px;">Borde rojo con relleno</p>

<p style="background-color: #f0f0f0; color: green; padding: 20px; border: 1px solid gray;">Fondo gris, texto verde</p>
💡 Consejo: Usa <strong> para texto importante (no solo visual) y <em> para énfasis. Los lectores de pantalla lo reconocen.

Listas Desordenadas

Las listas desordenadas (<ul>) muestran elementos sin orden específico:

<ul>
    <li>Elemento 1</li>
    <li>Elemento 2</li>
    <li>Elemento 3</li>
</ul>

Resultado:

  • Elemento 1
  • Elemento 2
  • Elemento 3

Tipos de viñeta (type):

  • disc (por defecto) - Círculo sólido
  • circle - Círculo vacío
  • square - Cuadrado

Ejemplo: <ul type="square">

Listas Ordenadas

Las listas ordenadas (<ol>) muestran elementos en secuencia:

<ol>
    <li>Primer paso</li>
    <li>Segundo paso</li>
    <li>Tercer paso</li>
</ol>

Resultado:

  1. Primer paso
  2. Segundo paso
  3. Tercer paso

Atributos de <ol>:

  • type - Tipo de numeración: "1", "A", "a", "I", "i"
  • start - Número inicial: start="5"
  • reversed - Orden inverso

Ejemplo: <ol type="A" start="3">

Listas Anidadas

Puedes combinar listas ordenadas y desordenadas:

<ul>
    <li>Frutas
        <ul>
            <li>Manzana</li>
            <li>Plátano</li>
        </ul>
    </li>
    <li>Verduras
        <ol>
            <li>Lechuga</li>
            <li>Zanahoria</li>
        </ol>
    </li>
</ul>
💡 Consejo: Las listas anidadas son ideales para menús de navegación y estructura de contenido.

Contenido Colapsable (Details)

Las etiquetas <details> y <summary> permiten crear contenido interactivo:

<details>
    <summary>Haz clic para ver más</summary>
    <p>Este contenido está oculto y aparece al hacer clic.</p>
</details>

Casos de uso:

  • Preguntas frecuentes (FAQ)
  • Acordeones
  • Información adicional opcional
💡 Beneficio: Mejora la experiencia del usuario al mostrar información solo cuando la necesita.

Ejemplo: Página Personal Completa

Vamos a crear una página personal con todo lo aprendido:

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width">
    <title>Mi Página Personal</title>
</head>
<body>
    <header>
        <h1>Juan Pérez</h1>
    </header>
    <main>
        <section>
            <h2>Sobre Mí</h2>
            <p>Soy estudiante de Ingeniería en Sistemas...</p>
        </section>
        <section>
            <h2>Mis Gustos</h2>
            <ul>
                <li>Programación</li>
                <li>Música</li>
            </ul>
        </section>
    </main>
    <footer>
        <p>© 2024 Juan Pérez</p>
    </footer>
</body>
</html>

Buenas Prácticas de Código

Un código limpio y bien estructurado es más fácil de mantener:

✅ Hacer:

  • Usar indentación (2 o 4 espacios)
  • Nombrar archivos en minúsculas
  • Agregar comentarios descriptivos
  • Cerrar todas las etiquetas
  • Usar rutas relativas

❌ Evitar:

  • Saltarse niveles de encabezados
  • Usar mayúsculas en etiquetas
  • Olvidar el atributo lang
  • No agregar meta viewport

Validación de Código HTML

El W3C (World Wide Web Consortium) ofrece un validador gratuito:

🔗 W3C Validator

URL: validator.w3.org

Permite validar tu código HTML por URL, archivo o texto directo.

Errores comunes:

  • Etiquetas sin cerrar
  • Atributos sin valor
  • Doctype incorrecto
  • Elementos mal anidados
💡 Consejo: Valida tu código frecuentemente durante el desarrollo. Es más fácil corregir errores pequeños que grandes.

Herramientas de Desarrollo

Los navegadores incluyen herramientas para inspeccionar y depurar:

🔧 DevTools (F12)

  • Elements: Ver estructura HTML y estilos
  • Console: Ver errores de JavaScript
  • Network: Analizar cargas de página
  • Inspector: Seleccionar elementos visuales

⌨️ Atajos útiles en VS Code:

  • Ctrl + / - Comentar/Descomentar
  • Ctrl + D - Seleccionar siguiente igual
  • Alt + ↑/↓ - Mover línea
  • Ctrl + Shift + K - Eliminar línea
