Qué son las Funciones

Las funciones son bloques de código reutilizables que realizan una tarea específica.


  • Evitan código repetido
  • Facilitan el mantenimiento
  • Permiten modularizar
  • Pueden recibir parámetros
  • Pueden retornar valores

Declaración de Funciones

// Función tradicional (declaración)
function saludar(nombre) {
    return "Hola, " + nombre;
}

// Llamar a la función
let mensaje = saludar("Juan");
alert(mensaje); // "Hola, Juan"
Hoisting: Las funciones declaradas con function pueden llamarse antes de su declaración.

Expresión de Función y Arrow

// Expresión de función
let saludar = function(nombre) {
    return "Hola, " + nombre;
};

// Arrow function (sintaxis moderna)
let saludar = (nombre) => "Hola, " + nombre;

// Arrow con cuerpo
let saludar = (nombre) => {
    return "Hola, " + nombre;
};
Arrow functions: Sintaxis más corta. Si tienen una sola línea, omiten return y llaves.

Parámetros y Retorno

// Múltiples parámetros
function sumar(a, b) {
    return a + b;
}

// Valor por defecto (ES6)
function saludar(nombre = "Usuario") {
    return "Hola, " + nombre;
}

// Sin retorno (undefined)
function mostrarMensaje(msg) {
    alert(msg);
}

  • return devuelve un valor y termina la función
  • Parámetros pueden tener valores por defecto

Ámbito de Variables (Scope)

let global = "Accesible en todas partes";

function mostrar() {
    let local = "Solo dentro de la función";
    console.log(global); // correcto
    console.log(local);  // correcto
}

console.log(global); // correcto
console.log(local);  // ERROR: no está definida
Regla: Las variables declaradas dentro de una función solo existen dentro de ella. let y const tienen ámbito de bloque.

Callbacks: Funciones como Argumento

// Función que recibe otra función
function procesar(dato, callback) {
    let resultado = dato * 2;
    callback(resultado);
}

// Llamada con arrow function
procesar(5, (valor) => {
    console.log("Resultado: " + valor);
});

// También con función nombrada
function mostrarResultado(valor) {
    alert("El doble es: " + valor);
}
procesar(5, mostrarResultado);

Buenas Prácticas con Funciones

  • Responsabilidad única: cada función hace una sola cosa
  • Nombres descriptivos: calcularPromedio(), no calc()
  • Evitar efectos secundarios: no modificar variables globales
  • Validar parámetros: verificar que los datos sean correctos
  • Usar const para funciones expresadas
