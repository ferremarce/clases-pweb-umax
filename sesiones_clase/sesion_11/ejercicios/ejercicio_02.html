Ejercicio 2: Gestor de Tareas v3

Sesión 11 - JavaScript III

Prof. Ing. Juan M. Ferreira

Enunciado

Refactoriza el Gestor de Tareas de la sesión anterior organizando el código con funciones. El programa sigue usando prompt(), alert() y un bucle while con menú.

  1. Función agregarTarea(tareas, nombre) — agrega una tarea al array
  2. Función listarTareas(tareas, callback) — recorre el array y ejecuta un callback por cada tarea
  3. Función completarTarea(tareas, indice, callback) — marca como completada y ejecuta un callback de notificación
  4. Función mostrarResumen(tareas) — muestra total de tareas y cuántas están completadas
  5. Función notificar(mensaje) — callback que muestra un alert()

Requisitos

  • Usar arrow functions para las funciones simples
  • Usar callbacks para notificaciones y listado
  • Mantener el bucle while con switch del menú
  • Cada tarea debe ser un objeto: { id, nombre, completada }
  • Validar que el nombre no esté vacío al agregar

Pistas

Ver pistas
  • const agregarTarea = (tareas, nombre) => { ... }
  • listarTareas(tareas, t => alert(t.nombre + " - " + (t.completada ? "Hecho" : "Pendiente")));
  • completarTarea(tareas, i, notificar);
  • tareas.length para total, .filter(t => t.completada).length para completadas
  • Revisa la solución de la sesión 10 como punto de partida

Resultado Esperado

  • Menú funcional con las mismas opciones que el v2
  • El código está organizado en funciones claras y reutilizables
  • Las notificaciones se manejan mediante callbacks
  • Se usa arrow functions para al menos 2 funciones
