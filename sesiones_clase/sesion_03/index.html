📌 Nota: Los enlaces e imágenes son fundamentales para la navegación y experiencia del usuario en la web.

Rutas de Archivos

Las rutas definen la ubicación de los recursos (imágenes, páginas, etc.):

Ruta Absoluta

Define la ubicación completa desde el dominio:

<img src="https://ejemplo.com/images/foto.jpg">

Usar: Solo para recursos externos (CDN, APIs)

Ruta Relativa

Define la ubicación desde el archivo actual:

<img src="images/foto.jpg">      <!-- misma carpeta -->
<img src="../images/foto.jpg">   <!-- carpeta padre -->
<img src="../../images/foto.jpg"> <!-- dos niveles arriba -->

Usar: Siempre que sea posible (recursos locales)

⚠️ Regla de oro: Usa rutas relativas para tus propios archivos. Son más portables y funcionan en cualquier servidor.

La Etiqueta de Enlace <a>

La etiqueta <a> (anchor) crea enlaces a otras páginas o recursos:

<a href="https://google.com">Ir a Google</a>
<a href="pagina.html">Nuestra Página</a>
<a href="archivo.pdf">Descargar PDF</a>

Atributos principales de <a>:

  • href: URL o ruta del destino
  • target: Dónde abrir el enlace
  • title: Texto emergente (tooltip)

Atributo Target

El atributo target especifica dónde abrir el enlace:

Valores de target:

  • _self (valor por defecto) - Abre en la misma pestaña
  • _blank - Abre en una nueva pestaña
  • _parent - Abre en el marco padre
  • _top - Abre en la ventana completa
<!-- Abre en nueva pestaña -->
<a href="https://google.com" target="_blank">Google</a>

<!-- Abre en la misma pestaña (por defecto) -->
<a href="pagina.html" target="_self">Nuestra Página</a>
💡 Consejo: Usa siempre target="_blank" para enlaces externos, así el usuario no pierde tu página.

Enlaces Internos (Anclas)

Los enlaces internos permiten navegar dentro de la misma página:

<!-- Definir el ancla -->
<h2 id="seccion1">Sección 1</h2>

<!-- Enlazar al ancla -->
<a href="#seccion1">Ir a Sección 1</a>

Ejemplo de menú de navegación:

<nav>
    <a href="#inicio">Inicio</a>
    <a href="#nosotros">Nosotros</a>
    <a href="#servicios">Servicios</a>
    <a href="#contacto">Contacto</a>
</nav>

La Etiqueta de Imagen <img>

La etiqueta <img> inserta imágenes en la página:

<img src="images/foto.jpg" 
       alt="Descripción de la imagen"
       width="400"
       height="300">

Atributos esenciales:

  • src: Ruta de la imagen (obligatorio)
  • alt: Texto alternativo (obligatorio para accesibilidad)
  • width: Ancho en píxeles
  • height: Alto en píxeles
⚠️ Importante: El atributo alt es obligatorio. Describe la imagen para lectores de pantalla y SEO.

Formatos de Imagen

Conoce los formatos más utilizados en la web:

🖼️ JPG/JPEG

Ideal para fotografías. Compresión con pérdida. No transparencias.

🖼️ PNG

Para gráficos con transparencias. Compresión sin pérdida. Fondos transparentes.

🎯 SVG

Gráficos vectoriales. Escalables sin pérdida. Ideal para iconos y logos.

🌐 WebP

Formato moderno. Mejor compresión que JPG y PNG. Soporte universal.

💡 Consejo: Usa WebP cuando sea posible (menor tamaño, misma calidad). Ten versiones PNG de respaldo.

Imágenes Responsivas

Las imágenes deben adaptarse a diferentes tamaños de pantalla:

Consejo básico:

Usa CSS para hacer imágenes flexibles:

img {
    max-width: 100%;
    height: auto;
}

Atributo srcset (avanzado):

<img src="imagen-grande.jpg"
       srcset="imagen-chica.jpg 480w, imagen-mediana.jpg 800w, imagen-grande.jpg 1200w"
       alt="Descripción">

El navegador elige la mejor versión según el dispositivo.

Audio en HTML

La etiqueta <audio> permite reproducir sonido:

<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
    <source src="audio.ogg" type="audio/ogg">
    Tu navegador no soporta audio.
</audio>

Atributos de <audio>:

  • controls: Muestra los controles de reproducción
  • autoplay: Reproduce automáticamente
  • loop: Repite infinitamente
  • muted: Silenciado al inicio

Video en HTML

La etiqueta <video> permite reproducir video:

<video controls width="640">
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">
    Tu navegador no soporta video.
</video>

Atributos de <video>:

  • controls: Muestra los controles
  • width/height: Dimensiones
  • poster: Imagen de portada
  • autoplay, loop, muted: Comportamiento
💡 Formatos recomendados: MP4 (H.264) + WebM. Siempre proporciona ambos para compatibilidad.

Iframes

Los iframes permiten incrustar contenido externo:

YouTube:

<iframe width="560" height="315"
          src="https://www.youtube.com/embed/VIDEO_ID"
          frameborder="0"
          allowfullscreen>
</iframe>

Google Maps:

<iframe src="https://www.google.com/maps/embed?..."
          width="600" height="450"
          style="border:0;"
          allowfullscreen>
</iframe>

Barra de Navegación

La navbar es el menú principal de un sitio web:

<nav>
    <a href="index.html">Inicio</a>
    <a href="nosotros.html">Nosotros</a>
    <a href="servicios.html">Servicios</a>
    <a href="contacto.html">Contacto</a>
</nav>

Mejores prácticas:

  • Usar la etiqueta semántica <nav>
  • Mantener enlaces claros y organizados
  • Alta contraste para accesibilidad
  • Testear en dispositivos móviles

Ejemplo: Página Completa con Multimedia

Veamos una página que integra todos los elementos:

<header>
    <h1>Mi Sitio de Viajes</h1>
    <nav>
        <a href="#inicio">Inicio</a>
        <a href="#destinos">Destinos</a>
        <a href="#videos">Videos</a>
    </nav>
</header>

<main>
    <section id="destinos">
        <img src="images/playa.jpg" alt="Playa Tropical">
    </section>
    
    <section id="videos">
        <video controls>
            <source src="videos/travel.mp4" type="video/mp4">
        </video>
    </section>
</main>

Buenas Prácticas

✅ Hacer:

  • Usar rutas relativas para recursos locales
  • Agregar siempre atributo alt a imágenes
  • Proporcionar múltiples formatos de video
  • Usar target="_blank" para enlaces externos
  • Comprimir imágenes antes de subirlas

❌ Evitar:

  • Rutas absolutas innecesarias
  • Imágenes sin texto alternativo
  • Videos con autoplay con sonido
  • Archivos de imagen muy grandes
  • Iframes sin permisos o contexto

Accesibilidad en Medios

Garantiza que todos puedan acceder al contenido:

Imágenes:

  • Siempre usar alt descriptivo
  • alt="" para imágenes decorativas
  • No usar imágenes de texto

Video y Audio:

  • Proporcionar subtítulos (track)
  • Transcripciones de audio
  • Controles accesibles
💡 Herramientas: Usa Lighthouse de Chrome para auditar accesibilidad.
