Proyecto Final I

Planificación y Maquetación

Objetivos de la Sesión

Al finalizar esta sesión, serás capaz de:

  • Seleccionar un proyecto web viable para el curso
  • Crear wireframes y mapa de sitio del proyecto
  • Inicializar un repositorio Git y conectar con GitHub
  • Maquetar la estructura HTML semántica del proyecto
  • Aplicar Bootstrap Grid para el layout responsivo
Repaso: En sesiones anteriores aprendimos HTML semántico, CSS responsivo, JavaScript (variables, funciones, DOM, eventos, localStorage) y Bootstrap (Grid y componentes).

Qué es un Proyecto Web

Un proyecto web integra todos los conocimientos del curso en un sitio funcional.


  • HTML semántico: estructura clara y accesible
  • CSS + Bootstrap: diseño atractivo y responsivo
  • JavaScript: interactividad y funcionalidades dinámicas
  • Git/GitHub: control de versiones y publicación

Opciones de Proyecto

Portafolio Personal

Galería de proyectos, formulario de contacto, filtros con JS.


Blog / Guía de Viajes

Tarjetas de destinos, tabla de precios, mapa interactivo.

Tienda / Catálogo

Productos, carrito con localStorage, checkout simulado.


Comunidad / Club

Noticias, galería, registro con validación, modales.

Importante: El proyecto se desarrolla en grupos de 3 a 5 integrantes. Todos deben contribuir al repositorio Git.

Metodología de Trabajo

  1. Formar grupos (3-5 personas) y elegir proyecto
  2. Diseñar wireframes y mapa de sitio
  3. Maquetar estructura HTML + Bootstrap
  4. Agregar interactividad con JavaScript (sesión 16)
  5. Publicar en GitHub Pages
  6. Presentar el proyecto final

Wireframes

Un wireframe es un esquema visual de la estructura de una página web.


  • Define la disposición de los elementos
  • No incluye colores ni estilos
  • Se enfoca en la funcionalidad y jerarquía
  • Herramientas: papel y lápiz, Figma, Balsamiq, draw.io
Actividad: Dibujar el wireframe de la página principal del proyecto en papel o herramienta digital.

Mapa de Sitio

El mapa de sitio muestra la estructura de navegación del proyecto.


mi-proyecto/
├── index.html          (Inicio)
├── acerca.html         (Acerca de)
├── servicios.html      (Servicios / Productos)
├── contacto.html       (Formulario de contacto)
└── proyecto.html       (Detalle de proyecto)
Consejo: Limitar a 4-6 páginas para un proyecto realista en el tiempo disponible.

Introducción a Git

Git es un sistema de control de versiones que permite:


  • Llevar un historial de cambios del código
  • Trabajar en equipo sin conflictos
  • Revertir cambios si algo falla
  • Publicar el proyecto en GitHub

Comandos Básicos de Git

# Configurar identidad
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

# Inicializar repositorio
git init

# Ver estado
git status

# Agregar archivos
git add .
git add index.html

# Hacer commit
git commit -m "feat: estructura inicial del proyecto"

GitHub: Crear Repositorio

  1. Crear cuenta en github.com
  2. Click en "New repository"
  3. Nombre del repo: mi-proyecto-web
  4. No inicializar con README
  5. Conectar repositorio local con GitHub:

git remote add origin https://github.com/usuario/mi-proyecto-web.git
git branch -M main
git push -u origin main

Maquetación con HTML Semántico

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Proyecto</title>
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">
    <link rel="stylesheet" href="css/style.css">
</head>
<body>
    <header>
        <nav>
            <!-- Barra de navegación -->
        </nav>
    </header>

    <main>
        <section>
            <!-- Contenido principal -->
        </section>
    </main>

    <footer>
        <!-- Pie de página -->
    </footer>

    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>
    <script src="js/script.js"></script>
</body>
</html>

Maquetación con Bootstrap Grid

<!-- Layout típico de proyecto -->
<div class="container">

    <div class="row py-5">

        </div>
        <div class="col-lg-4">
            <img src="images/hero.jpg" class="img-fluid" alt="Hero">
        </div>
    </div>

    <!-- Cards de contenido -->
    <div class="row row-cols-1 row-cols-md-3 g-4">
        <div class="col">
            <div class="card h-100">
                <div class="card-body">
                    <h5 class="card-title">Título</h5>
                    <p class="card-text">Contenido...</p>
                </div>
            </div>
        </div>
    </div>
</div>

Buenas Prácticas de Maquetación

  • Mobile-first: diseñar primero para móvil, luego ampliar
  • HTML semántico: header, nav, main, section, footer
  • Alt en imágenes: siempre descriptivo y accesible
  • CSS externo: separar estilos del HTML
  • Comentarios en equipo: coordinar cambios con Git
  • Commits frecuentes: un cambio lógico por commit
