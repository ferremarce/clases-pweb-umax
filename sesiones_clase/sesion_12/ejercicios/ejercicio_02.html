Ejercicio 2: Gestor de Tareas v4 con DOM

Sesión 12 - JavaScript IV

Prof. Ing. Juan M. Ferreira

Enunciado

Crea un gestor de tareas con interfaz visual. Los datos se guardan en un array en memoria (sin localStorage). Evoluciona el gestor de la sesión anterior usando manipulación del DOM.

  1. Un <input> para escribir la tarea y un botón "Agregar"
  2. Las tareas se muestran en una lista <ul> generada dinámicamente
  3. Cada tarea tiene botones "Completar" y "Eliminar"
  4. Al completar una tarea, se le agrega la clase CSS completada (tachado visual)
  5. Al eliminar, la tarea se borra del DOM y del array

Requisitos

  • Usar createElement() y appendChild() para crear la lista
  • Usar classList.toggle() o classList.add() para marcar completada
  • Usar remove() o removeChild() para eliminar tareas
  • Mantener un array interno con objetos { id, nombre, completada }
  • Función renderizarTareas() que actualiza el DOM desde el array

Pistas

Ver pistas
  • let li = document.createElement("li");
  • lista.appendChild(li);
  • li.classList.toggle("completada");
  • tareas.splice(indice, 1); para eliminar del array
  • Usa forEach() para recorrer tareas y generar el HTML

Resultado Esperado

  • Interfaz con input, botón Agregar y lista de tareas
  • Al agregar, aparece una nueva tarea en la lista
  • Al hacer clic en "Completar", la tarea se tacha visualmente
  • Al hacer clic en "Eliminar", la tarea desaparece
  • No se usa localStorage (los datos se pierden al recargar)
