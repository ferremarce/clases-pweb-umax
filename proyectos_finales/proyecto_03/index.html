Proyecto Final 3: Tablero Kanban de Tareas

INI04 - Introduccion al Desarrollo Web

Prof. Ing. Juan M. Ferreira

Descripcion General

Desarrollar un tablero Kanban tipo Trello con tres columnas (Pendiente, En Progreso, Completado) donde se pueden crear, mover y eliminar tarjetas de tareas. El proyecto integra HTML semantico, CSS responsivo con Grid y Flexbox, y JavaScript con manipulacion del DOM.

Duracion estimada: 3-4 sesiones de clase
Modalidad: Grupos de 3 estudiantes
Tecnologias: HTML5, CSS3, JavaScript (ES6+)

Requisitos HTML

  1. Estructura semantica completa: <header>, <nav>, <main>, <section>, <footer>
  2. Barra de navegacion fija con enlaces internos (anclas) a cada seccion
  3. Formulario para crear tareas con campos: titulo (text), descripcion (textarea), prioridad (select: Alta, Media, Baja), fecha limite (date)
  4. Tres columnas/secciones para los estados (Pendiente, En Progreso, Completado), cada una con encabezado y contador de tareas
  5. Cada tarjeta de tarea muestra: titulo, descripcion, prioridad (badge con color), fecha limite, botones de accion
  6. Validacion HTML5 en el formulario: required, minlength
  7. Uso de <fieldset> y <legend> para agrupar campos del formulario
  8. Seccion de filtros con botones para filtrar por prioridad
  9. Footer con nombres de los integrantes del equipo
  10. Atributo alt descriptivo en todas las imagenes

Requisitos CSS

  1. Variables CSS (:root) para la paleta de colores del proyecto y colores por prioridad
  2. CSS Grid para las 3 columnas del tablero: grid-template-columns: repeat(3, 1fr), apilandose a 1 columna en movil con media queries
  3. Flexbox dentro de cada columna para organizar las tarjetas verticalmente
  4. Diseno mobile-first con media queries (breakpoints: 576px, 768px, 992px)
  5. Tarjetas con box-shadow, border-radius y transiciones en hover
  6. Colores por prioridad: rojo para Alta, naranja para Media, verde para Baja (usando variables CSS)
  7. Badge de prioridad con position: absolute sobre tarjeta con position: relative
  8. Estados interactivos: :hover en tarjetas y botones, :focus en inputs
  9. Formulario estilizado con estados :valid e :invalid
  10. Navbar fija con position: fixed y z-index

Requisitos JavaScript

  1. Array de tareas con estructura: { id, titulo, descripcion, prioridad, fechaLimite, estado } y estados "pendiente", "en-progreso", "completado"
  2. Funcion que valida y agrega una tarea al array con estado "pendiente"
  3. Funcion que elimina una tarea del array
  4. Funcion que cambia el estado de una tarea en el array
  5. Funcion que filtra tareas por prioridad y re-renderiza
  6. Funcion que genera dinamicamente las tarjetas en cada columna con createElement
  7. Funcion que actualiza el numero de tareas en cada columna
  8. Funcion que retorna las tareas filtradas por estado
  9. Buenas practicas: eventos con addEventListener, uso de const/let, arrow functions y template literals
  10. Los datos se manejan en memoria (arrays), no se requiere localStorage

Resultado Esperado

  • La pagina carga mostrando el tablero con 3 columnas vacias y el formulario de creacion
  • Al crear una tarea, aparece como tarjeta en la columna "Pendiente"
  • Cada tarjeta muestra: titulo, descripcion, badge de prioridad con color, fecha limite
  • Los contadores de cada columna se actualizan al agregar/eliminar/mover tareas
  • Se puede mover una tarea a "En Progreso" o "Completado" con botones en la tarjeta
  • Se puede eliminar una tarea con un boton de eliminar
  • Los filtros por prioridad muestran solo las tareas de la prioridad seleccionada en todas las columnas
  • Las tareas completadas se muestran con un estilo visual diferente (tachado o opacidad reducida)
  • La pagina es completamente responsiva: en movil las columnas se apilan verticalmente
  • El codigo esta organizado, indentado y sigue las convenciones del curso (camelCase en JS, snake_case en CSS)

Entrega

  1. Formato: Carpeta comprimida (.zip) con la estructura de archivos indicada
  2. Estructura minima de archivos:
    • index.html - Pagina principal
    • css/estilos.css - Hoja de estilos
    • js/app.js - Logica de la aplicacion
    • images/ - Carpeta de imagenes (si aplica)
  3. Defensa oral: Cada integrante debe explicar una parte del proyecto y responder preguntas del docente
  4. Fecha limite: A definir por el docente
