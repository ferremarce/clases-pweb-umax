Proyecto Final 2: Sistema de Gestion de Notas

INI04 - Introduccion al Desarrollo Web

Prof. Ing. Juan M. Ferreira

Descripcion General

Desarrollar un sistema de gestion de notas estudiantil que permita a un profesor registrar estudiantes, asignar calificaciones por asignatura y generar reportes automaticos con estadisticas. El proyecto integra HTML semantico, CSS responsivo y JavaScript con manipulacion del DOM.

Duracion estimada: 3-4 sesiones de clase
Modalidad: Grupos de 3 estudiantes
Tecnologias: HTML5, CSS3, JavaScript (ES6+)

Requisitos HTML

  1. Estructura semantica completa: <header>, <nav>, <main>, <section>, <footer>
  2. Barra de navegacion fija con enlaces internos (anclas) a cada seccion
  3. Formulario de registro de estudiantes con campos: nombre completo, codigo de estudiante, email
  4. Formulario de registro de notas con: selector de estudiante (select), selector de asignatura (select), calificacion (input number con min=0, max=100)
  5. Tabla semantica con <thead>, <tbody>, <tfoot> para mostrar estudiantes y sus notas
  6. Seccion de reportes con <details> y <summary> para mostrar estadisticas
  7. Validacion HTML5 en formularios: required, type="email", minlength, min, max
  8. Uso de <fieldset> y <legend> para agrupar campos
  9. Footer con nombres de los integrantes del equipo
  10. Atributo alt descriptivo en todas las imagenes

Requisitos CSS

  1. Variables CSS (:root) para la paleta de colores del proyecto
  2. Flexbox para la navbar, formularios y layout general
  3. Diseno mobile-first con media queries (breakpoints: 576px, 768px, 992px)
  4. Estados interactivos: :hover en botones y filas de tabla, :focus en inputs
  5. Tabla responsiva con zebra striping (nth-child(even)) y hover en filas
  6. Formularios estilizados con estados :valid e :invalid
  7. Navbar fija con position: fixed y z-index
  8. Tarjetas de estadisticas con box-shadow y border-radius
  9. Colores condicionales: verde para aprobados (>=51), rojo para reprobados (<51)
  10. Transiciones suaves (transition) en botones y elementos interactivos

Requisitos JavaScript

  1. Array de estudiantes con estructura: { id, nombre, codigo, email, notas: [] }
  2. Funcion que valida y agrega un estudiante al array
  3. Funcion que elimina un estudiante y sus notas del array
  4. Funcion que agrega una nota al estudiante
  5. Funcion que retorna el promedio de un array de notas
  6. Funciones que retornan la nota mas alta y mas baja del array
  7. Funcion que calcula estadisticas generales (promedio general, % aprobados, mejor estudiante, peor estudiante)
  8. Funcion que genera dinamicamente las filas de la tabla con createElement
  9. Buenas practicas: eventos con addEventListener, uso de const/let, arrow functions y template literals
  10. Los datos se manejan en memoria (arrays), no se requiere localStorage

Resultado Esperado

  • La pagina carga mostrando formularios de registro y una tabla vacia
  • Al registrar un estudiante, aparece en la tabla y en el selector del formulario de notas
  • Al agregar una nota, se actualiza la tabla mostrando la nueva calificacion
  • Se puede eliminar estudiantes y sus notas desaparecen de la tabla
  • El pie de la tabla (tfoot) muestra el promedio general de todos los estudiantes
  • La seccion de reportes muestra: promedio general, cantidad de aprobados/reprobados, mejor y peor estudiante
  • Las notas se muestran con colores: verde si >= 51, rojo si < 51
  • La pagina es completamente responsiva: se adapta a movil, tablet y escritorio
  • El codigo esta organizado, indentado y sigue las convenciones del curso (camelCase en JS, snake_case en CSS)
