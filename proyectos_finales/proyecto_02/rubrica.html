Criterios de Evaluacion

La siguiente rubrica se utiliza para evaluar el Proyecto Final 2. Cada criterio tiene un peso porcentual que refleja su importancia relativa en la calificacion final.

Criterio Excelente (10) Bueno (7-9) Suficiente (4-6) Insuficiente (1-3)
HTML Semantico 20% Excelente (10) Estructura semantica completa y correcta. Uso adecuado de header, nav, main, section, footer. Formularios con label, fieldset, validacion HTML5. Tabla con thead, tbody, tfoot correctamente estructurados. Bueno (7-9) Estructura semantica mayormente correcta. Falta alguna etiqueta semantica menor. Formularios funcionales con validacion basica. Tabla con estructura correcta pero sin tfoot o detalles menores. Suficiente (4-6) Estructura HTML basica con uso limitado de etiquetas semanticas. Formularios sin validacion HTML5 o con errores en labels. Tabla sin estructura semantica completa. Insuficiente (1-3) HTML sin estructura semantica. Uso de div para todo. Formularios incompletos o sin labels. Tabla sin thead/tbody/tfoot.
CSS Responsivo 20% Excelente (10) Diseno mobile-first con media queries correctas. Uso eficaz de Flexbox. Variables CSS, transiciones, estados hover/focus. Tabla responsiva con zebra striping. Colores condicionales para aprobados/reprobados. Bueno (7-9) Diseno responsivo funcional con Flexbox. Media queries presentes pero con mejoras posibles. Variables CSS utilizadas. Estados interactivos basicos. Tabla responsiva. Suficiente (4-6) Diseno parcialmente responsivo. Uso basico de Flexbox sin media queries adecuadas. CSS sin variables. Pocos estados interactivos. Tabla no se adapta a movil. Insuficiente (1-3) Diseno no responsivo. No usa Flexbox. CSS desorganizado sin variables. No se adapta a diferentes pantallas.
JavaScript Funcional 25% Excelente (10) Todas las funciones requeridas implementadas correctamente. Calculos de promedio, maximo, minimo funcionan sin errores. Uso de const/let, arrow functions, template literals. Arrays con reduce, map, filter, find. Codigo limpio y bien organizado. Bueno (7-9) Mayoria de funciones implementadas. Calculos correctos con errores menores. Uso correcto de ES6+ con mejoras posibles. Logica funcional pero con codigo mejorable. Suficiente (4-6) Funciones basicas implementadas parcialmente. Mezcla de var con let/const. Calculos con errores menores que no impiden el funcionamiento basico. Insuficiente (1-3) Funciones incompletas o con errores graves. Uso de var. Codigo desorganizado. Los calculos no funcionan correctamente.
Manipulacion del DOM 20% Excelente (10) Renderizado dinamico completo de tabla con createElement. Eventos con addEventListener (sin onclick inline). Uso de classList, textContent, innerHTML. Actualizacion dinamica de tabla, selectores y reportes. Colores condicionales aplicados dinamicamente. Bueno (7-9) Renderizado dinamico funcional. Eventos registrados correctamente. Uso adecuado de classList. Algunas actualizaciones del DOM podrian optimizarse. Suficiente (4-6) Manipulacion basica del DOM. Mezcla de onclick inline con addEventListener. Renderizado parcial. Uso excesivo de innerHTML. Insuficiente (1-3) Poca o nula manipulacion dinamica del DOM. Todo el contenido es estatico en HTML. No usa addEventListener. No hay interactividad real.
Organizacion y Trabajo en Equipo 15% Excelente (10) Codigo bien organizado y comentado. Nombres descriptivos. Estructura de archivos correcta. Todos los integrantes dominan todo el proyecto en la defensa oral. Bueno (7-9) Codigo organizado con nombres claros. Estructura de archivos correcta. Los integrantes conocen bien su parte y tienen nocion general del proyecto. Suficiente (4-6) Codigo parcialmente organizado. Nombres poco descriptivos. Estructura de archivos basica. Los integrantes solo conocen su parte especifica. Insuficiente (1-3) Codigo desorganizado sin convenciones. Estructura de archivos incorrecta. Los integrantes no pueden explicar el proyecto en la defensa oral.
