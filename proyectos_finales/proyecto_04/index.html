Proyecto Final 4: Gestor de Gastos Personales

INI04 - Introduccion al Desarrollo Web

Prof. Ing. Juan M. Ferreira

Descripcion General

Desarrollar una aplicacion de control financiero personal que permita registrar ingresos y gastos, categorizarlos, visualizar balances por categoria y filtrar por tipo o periodo. El proyecto integra HTML semantico, CSS responsivo con Grid y Flexbox, y JavaScript con manipulacion del DOM.

Duracion estimada: 3-4 sesiones de clase
Modalidad: Grupos de 3 estudiantes
Tecnologias: HTML5, CSS3, JavaScript (ES6+)

Requisitos HTML

  1. Estructura semantica completa: <header>, <nav>, <main>, <section>, <footer>
  2. Barra de navegacion fija con enlaces internos (anclas) a cada seccion
  3. Formulario de registro con campos: tipo (radio buttons: Ingreso/Gasto), monto (number con min=0), categoria (select dinamico segun tipo), descripcion (text), fecha (date)
  4. Seccion de resumen con 3 tarjetas (Total Ingresos, Total Gastos, Balance) y seccion de filtros por tipo y categoria
  5. Tabla semantica con <thead>, <tbody>, <tfoot> para mostrar el historial de transacciones
  6. Validacion HTML5 en el formulario: required, min="0", minlength
  7. Uso de <fieldset> y <legend> para agrupar campos del formulario
  8. Seccion de desglose por categoria con <details> y <summary>
  9. Footer con nombres de los integrantes del equipo
  10. Atributo alt descriptivo en todas las imagenes

Requisitos CSS

  1. Variables CSS (:root) para la paleta de colores del proyecto, incluyendo colores para ingresos (verde) y gastos (rojo)
  2. CSS Grid para las 3 tarjetas de resumen: grid-template-columns: repeat(3, 1fr), apilandose a 1 columna en movil
  3. Flexbox para la navbar, formulario, filtros y layout general
  4. Diseno mobile-first con media queries (breakpoints: 576px, 768px, 992px)
  5. Colores condicionales: verde para filas de ingreso, rojo para filas de gasto
  6. Tabla responsiva con zebra striping (nth-child(even)) y hover en filas
  7. Estados interactivos: :hover en botones y filas, :focus en inputs
  8. Formulario estilizado con estados :valid e :invalid
  9. Navbar fija con position: fixed y z-index
  10. Tarjetas de resumen con box-shadow, border-radius y borde superior de color segun tipo

Requisitos JavaScript

  1. Objeto de categorias predefinidas { ingresos: [...], gastos: [...] } y array de transacciones con estructura { id, tipo, monto, categoria, descripcion, fecha }
  2. Funcion que valida y agrega una transaccion al array
  3. Funcion que elimina una transaccion del array
  4. Funciones que retornan el balance general y la suma por tipo
  5. Funcion que retorna un objeto con totales por cada categoria
  6. Funciones que filtran las transacciones por tipo y categoria
  7. Funcion que genera dinamicamente las filas de la tabla con createElement
  8. Funcion que actualiza las 3 tarjetas de resumen con los totales
  9. Funcion que cambia las opciones del select segun el tipo seleccionado
  10. Buenas practicas: eventos con addEventListener, uso de const/let, arrow functions, template literals y datos en memoria sin localStorage

Resultado Esperado

  • La pagina carga mostrando las 3 tarjetas de resumen en cero y una tabla vacia
  • Al seleccionar Ingreso o Gasto en el formulario, el select de categorias se actualiza automaticamente
  • Al registrar una transaccion, aparece en la tabla y las tarjetas de resumen se actualizan
  • Las filas de ingreso se muestran en verde y las de gasto en rojo
  • Se puede eliminar transacciones y los totales se recalculan automaticamente
  • Los filtros por tipo (Todos, Ingresos, Gastos) muestran solo las transacciones correspondientes
  • El filtro por categoria permite ver solo transacciones de una categoria especifica
  • La seccion de desglose por categoria muestra el total gastado/ingresado en cada categoria
  • La pagina es completamente responsiva: se adapta a movil, tablet y escritorio
  • El codigo esta organizado, indentado y sigue las convenciones del curso (camelCase en JS, snake_case en CSS)
