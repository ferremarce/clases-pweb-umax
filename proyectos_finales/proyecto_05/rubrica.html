Criterios de Evaluacion

La siguiente rubrica se utiliza para evaluar el Proyecto Final 5. Cada criterio tiene un peso porcentual que refleja su importancia relativa en la calificacion final.

Criterio Excelente (10) Bueno (7-9) Suficiente (4-6) Insuficiente (1-3)
HTML Semantico 20% Excelente (10) Estructura semantica completa y correcta. Uso adecuado de header, nav, main, section, footer. Tres pantallas bien diferenciadas. Formularios con fieldset/legend. Tabla de resultados con thead, tbody, tfoot. Bueno (7-9) Estructura semantica mayormente correcta. Tres pantallas presentes. Falta alguna etiqueta semantica menor. Formularios funcionales. Tabla con estructura correcta. Suficiente (4-6) Estructura HTML basica con uso limitado de etiquetas semanticas. Pantallas no bien diferenciadas. Formularios sin fieldset. Tabla sin estructura semantica completa. Insuficiente (1-3) HTML sin estructura semantica. Uso de div para todo. Pantallas mezcladas. Formularios incompletos. Tabla sin thead/tbody/tfoot.
CSS Responsivo 20% Excelente (10) Diseno mobile-first con media queries correctas. Uso eficaz de Flexbox. Variables CSS, transiciones, estados hover/checked. Barra de progreso dinamica. Colores de retroalimentacion. Animaciones de entrada/salida. Bueno (7-9) Diseno responsivo funcional con Flexbox. Media queries presentes pero con mejoras posibles. Variables CSS utilizadas. Estados interactivos basicos. Barra de progreso funcional. Suficiente (4-6) Diseno parcialmente responsivo. Uso basico de Flexbox sin media queries adecuadas. CSS sin variables. Pocos estados interactivos. Barra de progreso basica. Insuficiente (1-3) Diseno no responsivo. No usa Flexbox. CSS desorganizado sin variables. No se adapta a diferentes pantallas.
JavaScript Funcional 25% Excelente (10) Todas las funciones requeridas implementadas correctamente. Temporizador con setInterval/clearInterval funciona sin errores. Navegacion entre preguntas fluida. Calculo de puntuacion correcto. Uso de const/let, arrow functions, template literals. Bueno (7-9) Mayoria de funciones implementadas. Temporizador funcional con errores menores. Navegacion correcta. Calculos correctos. Uso correcto de ES6+ con mejoras posibles. Suficiente (4-6) Funciones basicas implementadas parcialmente. Temporizador con errores o no funcional. Mezcla de var con let/const. Navegacion con problemas menores. Insuficiente (1-3) Funciones incompletas o con errores graves. Temporizador no funciona. Uso de var. Navegacion rota. El quiz no funciona correctamente.
Manipulacion del DOM 20% Excelente (10) Renderizado dinamico completo de preguntas con createElement. Eventos con addEventListener (sin onclick inline). Uso de classList, textContent, innerHTML. Actualizacion dinamica de barra de progreso, temporizador y resultados. Retroalimentacion visual inmediata. Bueno (7-9) Renderizado dinamico funcional. Eventos registrados correctamente. Uso adecuado de classList. Algunas actualizaciones del DOM podrian optimizarse. Retroalimentacion presente. Suficiente (4-6) Manipulacion basica del DOM. Mezcla de onclick inline con addEventListener. Renderizado parcial. Uso excesivo de innerHTML. Retroalimentacion limitada. Insuficiente (1-3) Poca o nula manipulacion dinamica del DOM. Todo el contenido es estatico en HTML. No usa addEventListener. No hay interactividad real.
Organizacion y Trabajo en Equipo 15% Excelente (10) Codigo bien organizado y comentado. Nombres descriptivos. Estructura de archivos correcta. Todos los integrantes dominan todo el proyecto en la defensa oral. Bueno (7-9) Codigo organizado con nombres claros. Estructura de archivos correcta. Los integrantes conocen bien su parte y tienen nocion general del proyecto. Suficiente (4-6) Codigo parcialmente organizado. Nombres poco descriptivos. Estructura de archivos basica. Los integrantes solo conocen su parte especifica. Insuficiente (1-3) Codigo desorganizado sin convenciones. Estructura de archivos incorrecta. Los integrantes no pueden explicar el proyecto en la defensa oral.
